# ADR-0001: 用 CLIP 向量检索而非 VLM 生成 caption

- **状态**：Accepted
- **日期**：2026-07-10

## 背景与问题

LLaVA 等 VLM 能"看图说话"。一种思路是用 VLM 为每张图生成文字描述，再做文本检索。
是否该走这条路？

## 候选方案

- **方案 A**：CLIP/SigLIP 把图与文编码到同一向量空间，做近似最近邻检索。
- **方案 B**：VLM 生成 caption，落库后做全文/文本向量检索。

## 决策

采用方案 A：CLIP/SigLIP 向量检索。

## 理由

1. **延迟**：CLIP 编码一张图 ~50ms，VLM 生成 caption ~2-5s，差约 100x。
2. **信息损失**：caption 是有损压缩（语言无法穷尽画面细节），CLIP embedding 保留完整语义。
3. **资源**：CLIP ViT-B-16 ~400MB，LLaVA-7B ~14GB，NAS/单卡更适合前者。
4. **检索范式**：向量 ANN 是 O(log N)，语义匹配好；全文倒排对语义匹配弱。

## 后果

- **正面**：低延迟、低资源、语义保真、天然支持图搜图。
- **负面**：无法回答"图里写了什么字/几个人"这类需要理解的复杂问题——那是 VLM 的活，可在
  Phase 3 作为结果精排/问答的补充，不放进检索主链路。
