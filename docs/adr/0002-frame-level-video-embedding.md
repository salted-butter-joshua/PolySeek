# ADR-0002: 视频用帧级 CLIP 而非专门的视频 Embedding 模型

- **状态**：Accepted
- **日期**：2026-07-10

## 背景与问题

有 VideoCLIP、InternVideo2 等专门的视频 Embedding 模型。视频检索是否该用它们？

## 候选方案

- **方案 A**：ffmpeg 固定间隔抽帧，逐帧用图片 CLIP 编码，检索命中后按视频去重。
- **方案 B**：视频级 Embedding 模型直接编码整段视频。

## 决策

Phase 1/2 采用方案 A（帧级 CLIP），Phase 3 视需要再评估视频级模型。

## 理由

1. 帧级 CLIP 覆盖 90% "找特定画面"的需求。
2. 视频级模型通常更大（2-5GB）、更慢。
3. 帧级能精确定位到秒级时间戳，视频级只能到视频粒度。
4. 复用图片 Embedding 的全部代码，复杂度低。

## 后果

- **正面**：秒级定位、代码复用、资源省。
- **负面**：抽帧会漏掉快速运动/时序语义（"某人先坐下再起身"这类时序查询）。
- **后续影响**：抽帧间隔（默认 2s）是精度/成本的权衡旋钮；可后续加场景切换检测优化。
