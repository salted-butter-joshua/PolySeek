# ADR-0003: 音频走 Whisper 转写而非 CLAP 音频 Embedding

- **状态**：Accepted
- **日期**：2026-07-10

## 背景与问题

CLAP（Contrastive Language-Audio Pretraining）能直接做音频-文本对齐。音频检索用它还是走转写？

## 候选方案

- **方案 A**：Whisper 转写为文本分段 → 用 CLIP 文本编码器编码进同一空间。
- **方案 B**：CLAP 直接编码音频波形。

## 决策

采用方案 A：Whisper 转写 → 文本 Embedding。

## 理由

1. CLAP 训练数据以环境声（鸟叫/雨声/音乐）为主，对语音内容理解弱。
2. NAS 音频多为播客/录音/有声书，核心信息在**语音内容**而非声学特征。
3. Whisper 中文转写 WER < 10%，转文本后与图文同处一个 CLIP 空间，天然支持文/图搜音频。
4. 转写文本还能做全文检索 fallback，一举两得。

## 后果

- **正面**：语音语义检索准，复用文本编码，多一层全文检索能力。
- **负面**：纯音乐/环境声无法被有效检索（无语音内容）；转写有额外耗时（离线阶段最重的一环）。
- **后续影响**：若出现大量音乐类需求，可再引 CLAP 作为并行索引。
