# ADR-0004: Embedding 后端抽象 + 可插拔

- **状态**：Accepted
- **日期**：2026-07-10

## 背景与问题

中文场景 Chinese-CLIP 强，SigLIP 2 检索评分更可靠且多语言，OpenAI CLIP 是英文基线。
不同项目/数据分布最优后端不同，如何避免锁死一个模型？

## 候选方案

- **方案 A**：定义 `EmbeddingService` 抽象基类，各后端实现同一接口，工厂按 config 创建。
- **方案 B**：直接写死某一个后端。

## 决策

采用方案 A：抽象基类 + 工厂模式，`config.embedding.backend` 切换，重依赖延迟导入。

## 理由

1. 上层（ingestion/search）只依赖接口，换模型不改业务代码。
2. 各后端的重依赖（cn_clip / transformers）在各自模块内延迟 import，未选用的不必安装。
3. 便于做 A/B 对比（同一评测集比 Recall@K）。

## 后果

- **正面**：换模型零成本、依赖解耦、可对比。
- **负面**：不同后端向量维度不同（512/768/1024），**换后端需重建索引**（collection 维度绑定）。
- **约束**：所有后端统一输出 L2 归一化向量，使 Cosine=点积，向量库度量统一。
